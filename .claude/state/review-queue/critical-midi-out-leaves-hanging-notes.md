# Critical: MidiOut can leave hanging notes when transport or routing changes

`MidiOut` schedules note-offs in `pendingNoteOffs`, but nothing flushes that state when playback stops or when a track stops targeting the MIDI endpoint (`Sources/Engine/Blocks/MidiOut.swift:12-61`, `Sources/Engine/EngineController.swift:74-113`, `Sources/Engine/EngineController.swift:266-274`). If the transport is stopped mid-gate, or if a track is muted / rerouted to audio before its scheduled note-off tick arrives, the matching note-off is never emitted and the destination can hold the note indefinitely. This is a transport-level correctness bug, not just a polish issue. The controller needs an explicit "flush pending note-offs now" path when stopping playback and before detaching a MIDI output, with regression coverage proving the note-off is actually delivered.
