# Important: SeqAIDocumentModel can hold an impossible selectedTrackID

`SeqAIDocumentModel.init(version:tracks:selectedTrackID:)` stores `selectedTrackID` verbatim (`Sources/Document/SeqAIDocumentModel.swift:75-79`), even when that UUID does not exist in `tracks`. Every read then silently falls back to `tracks[0]` through `selectedTrackIndex` (`Sources/Document/SeqAIDocumentModel.swift:23-25`), so the model can say “selected track = X” while the UI/editor actually operates on track 0. That is already happening in `test_selected_au_output_routes_note_events_to_audio_sink`, which passes `StepSequenceTrack.default.id` for a newly created synth track and still passes because of the fallback (`Tests/SequencerAITests/Engine/EngineControllerTests.swift:89-106`). This is both a contract lie and a test that passes for the wrong reason: the initializer should normalize invalid IDs the same way `init(from:)` does, and the test should assert with the real track ID so selection bugs cannot hide behind the fallback path.
