# Live View Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a real `Live` workspace for performance editing over the current phrase/layer state. It should use the same fresh model as Phrase view: project-scoped layers, per-(phrase, track, layer) cells, and type-driven editing. It should operate over the flat track list and optionally collapse groups into aggregate cells.

**Architecture:** `LiveWorkspaceView` is a focused performance matrix. The transport bar owns `TransportMode` (`Song` / `Free`). The live surface itself chooses a layer, renders one cell per visible track (or group aggregate), and edits cell values according to the selected layer's value type:

- boolean layers → toggle / bar toggles
- indexed layers like `Pattern` → slot pickers
- scalar layers → drag, bars, steps, curve/ramp

There is no `trackSlots`, no `DrumTagCell`, and no per-tag live expansion. Drum kits are already represented as grouped mono tracks.

**Tech Stack:** Swift 5.9+, SwiftUI, Foundation, XCTest.

**Parent spec:** `/Users/maxwilliams/dev/sequencer-ai/docs/specs/2026-04-18-north-star-design.md`

**Status:** In progress. `TransportMode` and a basic `Live` shell already exist. This plan now focuses on making the live surface genuinely performance-oriented.
