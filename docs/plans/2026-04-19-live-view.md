# Live Performance View + Transport Modes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the app a **Live view** — a transport-aware matrix for direct performance editing over the current phrase/layer state. It should default to a gratifying mute-oriented view, switch layers at the top, and let the user work either in **song** mode (follow phrase progression) or **free** mode (pin the current phrase while the live grid drives the sound directly). A "Save as Phrase" action can come once the matrix itself is real.

**Architecture:** Reuses the current phrase/layer model and the new flat/grouped track model. `TransportMode` now lives on `EngineController`, exposed in the main transport bar. The live surface should render over the current flat track list, optionally with group aggregate cells, not over retired `trackSlots` or per-tag drum expansions. Cell controls should still be value-type driven the way the Phrase view is: toggles for boolean layers, scalar drags for continuous layers, indexed pickers for pattern selection.

## Reshape Note

This plan was originally written against slot-based layout and per-tag drum expansion. Those assumptions are now retired.

- `trackSlots` is gone; the live view should follow the flat track list, just like the Tracks matrix.
- `DrumTagCell` / "Expand Drums" is not the current direction. Drum voices are already flat tracks via `TrackGroup`.
- If a compact overview is needed, use a **group aggregate** cell or collapse toggle rather than per-tag expansion.
- `TransportMode` is already in the codebase as a real engine concept, so later tasks should build on that rather than reintroduce it.

**Tech Stack:** Swift 5.9+, SwiftUI (LazyVGrid, DragGesture, ContextMenu), Foundation, XCTest.

**Parent spec:** `docs/specs/2026-04-18-north-star-design.md` — §"UX surfaces" (Perform view's neighbour), §"Track types, patterns, and phrases" (phrase cell model reused), §"Pipeline layer" (mute is a concrete macro row). Live view isn't explicitly listed in the spec yet — this plan's §"Spec delta" at the bottom appends a short paragraph.

**Environment note:** Xcode 16. All `xcodebuild` invocations prefix `DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer`.

**Status:** <STATUS_PREFIX> <COMPLETED_MARKER> TBD. Tag `v0.0.8-live-view` at TBD.

**Depends on:**

- `2026-04-19-track-group-reshape.md` — live view should build on the flat/grouped track model.
- `2026-04-19-tracks-matrix.md` — the same matrix language and track selection model should be shared.
- A phrase/layer data foundation that supports layer-specific cell editing; if the document model still only stores per-track mode shells, that alignment work is a prerequisite.

**Deliberately deferred:**

- **Scene / snapshot recall** (MPC-style — save grid state as a "scene" restorable live). "Save as Phrase" covers the persistent version; runtime-only scenes land later.
- **XY pad / macro-vector control** — already covered by the existing Perform overlay. Live view is for per-track per-layer edits, not for abstract macro control.
- **Recording Live-view edits as a Take** — takes already exist per spec; wiring Live edits to take capture is a follow-up.
- **Mixer integration** — per-bus mute is Mixer's job. Live view should stay phrase/layer focused.
- **Per-voice drum expansion.** The fresh grouped model already flattened drum voices into tracks, so per-tag expansion is no longer an MVP requirement.

struct TrackCell: View {
    let track: StepSequenceTrack
    let layer: Layer
    let cellValue: CellValue
    let onEdit: (CellValue) -> Void

    var body: some View {
        Group {
            switch layer.valueType {
            case .boolean:
                BooleanToggleCell(value: cellValue.asBool, onToggle: { onEdit(.single(.bool($0))) })
            case .scalar(let range):
                ScalarDragCell(value: cellValue.asDouble, range: range, onChange: { onEdit(.single(.double($0))) })
            case .patternIndex:
                IndexedPickerCell(value: cellValue.asInt, maxIndex: 15, onPick: { onEdit(.single(.int($0))) })
            case .chord:
                ChordCell(chord: cellValue.asChord)
            case .enumTag(let tags):
                IndexedPickerCell(value: ..., labels: tags, onPick: ...)
            }
        }
        .overlay(alignment: .topLeading) {
            // Track-type glyph + name inset in corner so the cell still identifies the track
            HStack(spacing: 4) {
                TrackTypeGlyph(type: track.trackType).frame(width: 12, height: 12)
                Text(track.name)
                    .font(.system(size: 9, weight: .semibold))
                    .lineLimit(1)
            }
            .padding(4)
        }
        .accessibilityIdentifier("live-track-cell-\(track.id.uuidString)-\(layer.id)")
    }
}

struct DrumTagCell: View {
    let trackID: TrackID
    let tag: VoiceTag
    let isMuted: Bool                    // Mute-layer only for MVP
    let onToggle: () -> Void

    var body: some View {
        Button(action: onToggle) {
            VStack(spacing: 2) {
                Text(tag.prefix(6).uppercased())
                    .font(.system(size: 9, weight: .bold, design: .monospaced))
                    .lineLimit(1)
                Rectangle()
                    .fill(isMuted ? Color.red.opacity(0.7) : Color.green.opacity(0.7))
                    .frame(height: 6)
            }
            .frame(maxWidth: .infinity, maxHeight: .infinity)
            .padding(4)
            .background(StudioTheme.border.opacity(0.05))
            .overlay(
                RoundedRectangle(cornerRadius: 4)
                    .strokeBorder(StudioTheme.magenta.opacity(0.35),     // distinct from regular track cells
                                  style: StrokeStyle(lineWidth: 1, dash: [2, 1]))
            )
        }
        .buttonStyle(.plain)
        .accessibilityIdentifier("live-drum-tag-cell-\(trackID.uuidString)-\(tag)")
    }
}
```

public enum LiveDisplayCell: Equatable {
    case track(TrackID)
    case drumTag(TrackID, VoiceTag)
    case empty
}

public enum LiveLayout {
    /// Compute the flat display-cell sequence for the grid.
    /// In normal mode: one entry per trackSlot (track or empty).
    /// In drum-expanded mode: drum tracks' single entry is replaced by one entry per voice tag
    /// in track.voicing.destinations (sorted for stability). Non-drum tracks stay single-entry.
    /// Output is padded with .empty entries up to TracksLayout.slotCount.
    /// If the expanded sequence would exceed slotCount, truncates and reports overflow.
    public static func displayCells(
        trackSlots: [TrackID?],
        tracks: [TrackID: StepSequenceTrack],
        drumsExpanded: Bool
    ) -> (cells: [LiveDisplayCell], overflow: Int)
}
```

**View shape:**

```swift
struct LivePerformanceView: View {
    @Binding var document: SeqAIDocument
    @ObservedObject var engineController: EngineController
    @State private var activeLayerID: LayerID = .mute        // defaults to Mute
    @State private var drumsExpanded: Bool = false
    @State private var isSaveAsPhrasePresented = false

    private let columns: [GridItem] = Array(
        repeating: GridItem(.flexible(), spacing: 8),
        count: TracksLayout.columnCount
    )

    private var activePhrase: Phrase? {
        document.model.phrases[safe: engineController.currentPhraseIndex]
    }

    private var layout: (cells: [LiveDisplayCell], overflow: Int) {
        LiveLayout.displayCells(
            trackSlots: document.model.trackSlots,
            tracks: document.model.tracksByID,
            drumsExpanded: drumsExpanded && activeLayerID == .mute
        )
    }

    var body: some View {
        VStack(spacing: 12) {
            // Top bar: layer selector + drums-expand toggle (visible when on Mute)
            HStack {
                LayerSelectorBar(
                    layers: document.model.layers,
                    activeLayerID: $activeLayerID
                )
                Spacer()
                if activeLayerID == .mute {
                    Toggle("Expand Drums", isOn: $drumsExpanded)
                        .toggleStyle(.button)
                        .accessibilityIdentifier("expand-drums-toggle")
                }
            }
            .padding(.horizontal)

            if layout.overflow > 0 {
                // Warning strip
                Text("\(layout.overflow) cells hidden — too many tags to fit. Collapse drums or reduce tags.")
                    .font(.caption)
                    .foregroundStyle(.orange)
                    .padding(.horizontal)
            }

            LazyVGrid(columns: columns, spacing: 8) {
                ForEach(Array(layout.cells.enumerated()), id: \.offset) { _, cell in
                    switch cell {
                    case .track(let id):
                        if let track = document.model.track(withID: id),
                           let phrase = activePhrase,
                           let layer = document.model.layer(id: activeLayerID)
                        {
                            TrackCell(
                                track: track,
                                layer: layer,
                                cellValue: phrase.cellValue(for: track.id, layer: layer.id),
                                onEdit: { newValue in
                                    document.model.setCell(
                                        phraseID: phrase.id,
                                        trackID: track.id,
                                        layerID: layer.id,
                                        value: newValue
                                    )
                                }
                            )
                            .aspectRatio(1.0, contentMode: .fit)
                        }
                    case .drumTag(let trackID, let tag):
                        if let phrase = activePhrase,
                           let currentMutedTags = phrase.mutedTagSet(for: trackID)
                        {
                            DrumTagCell(
                                trackID: trackID,
                                tag: tag,
                                isMuted: currentMutedTags.contains(tag)
                            ) {
                                var newSet = currentMutedTags
                                if newSet.contains(tag) { newSet.remove(tag) } else { newSet.insert(tag) }
                                document.model.setCell(
                                    phraseID: phrase.id,
                                    trackID: trackID,
                                    layerID: .mute,
                                    value: .singleMuteTagSet(newSet)
                                )
                            }
                            .aspectRatio(1.0, contentMode: .fit)
                        }
                    case .empty:
                        Color.clear
                            .frame(maxWidth: .infinity, maxHeight: .infinity)
                            .aspectRatio(1.0, contentMode: .fit)
                    }
                }
            }
            .padding()

            HStack {
                Spacer()
                Button("Save as Phrase") { isSaveAsPhrasePresented = true }
                    .accessibilityIdentifier("save-as-phrase")
            }
            .padding(.horizontal)
        }
        .sheet(isPresented: $isSaveAsPhrasePresented) {
            SaveAsPhraseSheet(
                isPresented: $isSaveAsPhrasePresented,
                suggestedName: "Phrase \(document.model.phrases.count + 1)"
            ) { name in
                document.model.appendPhrase(copyingCellsFrom: engineController.currentPhraseIndex, name: name)
            }
        }
        .accessibilityIdentifier("live-performance-view")
    }
}
```

**Tests (LiveLayout):**

1. Normal mode: `displayCells` for 3 populated + 61 empty slots → 3 `.track(id)` + 61 `.empty`, overflow 0.
2. Drum-expanded, one drum track at slot 0 with 3 tags + 2 other tracks + 61 empty slots: first 3 cells are `.drumTag(drumID, tag)` (tags sorted alphabetically), next 2 are the other tracks, remaining `.empty`. Overflow 0.

**Type change:**

```swift
public enum CellValue: Codable, Equatable, Sendable {
    case inheritDefault
    case single(ScalarValue)                   // Bool / Double / Int / Chord payloads
    case bars([ScalarValue])
    case steps([ScalarValue])
    case curve(ControlPoints)
    case singleMuteTagSet(Set<VoiceTag>)       // NEW — Mute layer, drum tracks only
}

extension Phrase {
    public func mutedTagSet(for trackID: TrackID) -> Set<VoiceTag> {
        if case .singleMuteTagSet(let set) = cellValue(for: trackID, layer: .mute) {
            return set
        }
        return []
    }
}
```

Legacy decode: existing `.single(.bool(true))` cells on a Mute layer for a drum track stay valid — treated as "all tags muted"; `.single(.bool(false))` treated as "no tags muted". A migration pass on document open converts these into `.singleMuteTagSet` form with the appropriate set (all tags or empty) so subsequent edits operate on the new shape.

**Engine-side behaviour:**

- During each tick, after a drum track's block produces its note-stream, filter out note events whose `voiceTag` is in the active phrase's `mutedTagSet(for: trackID)`. Events with tag `nil` (untagged) pass through regardless.
- For non-drum tracks, the Mute layer still uses `.single(.bool)` and either drops ALL events (when muted) or none.

**Tests:**

1. `CellValue.singleMuteTagSet(["kick"])` round-trips through Codable.
2. `Phrase.mutedTagSet(for: drumTrackID)` returns the set when the cell is `.singleMuteTagSet`; returns empty when the cell is `.inheritDefault` or `.single(.bool(false))`.
3. Legacy decode: a document with `.single(.bool(true))` on a drum-track Mute cell decodes + migrates to `.singleMuteTagSet([all-tags])`.
4. Engine tick: drum track emits notes tagged `["kick", "snare", "hat"]`; phrase has `mutedTagSet = ["snare"]`; router dispatcher sees kick + hat events, not snare.
5. Non-drum track's Mute cell `.single(.bool(true))` drops every event (untouched behaviour).

## Open questions resolved for this plan

- **Default active layer:** `Mute`. Boot-into-gratifying-state: user opens Live view, sees per-track mute toggles, can start playing with them immediately.
- **Drum expansion scope:** Mute layer only. Toggle (`"Expand Drums"`) is a toolbar button visible only when `activeLayerID == .mute`. Hidden / non-functional on other layers.
- **Drum-expanded rendering:** tag cells become **first-class cells in the matrix**, distinct from the track cells — smaller font, dashed magenta border, monospace tag labels, prominent mute state bar. Visually obvious that "this cell is a tag, not a whole track".
- **Overflow handling:** if expanded-mode totals exceed 64 cells (`TracksLayout.slotCount`), the layout truncates and a warning strip appears — "N cells hidden — collapse drums or reduce tags." No scrolling in MVP; scrolling is a follow-up if the 64-cap feels tight in practice.
- **Tag ordering in expanded mode:** alphabetical by tag name (stable). Drum tracks with the same tag set render their cells in the same order every time. User can rename tags in drum track detail if the default "kick / snare / hat / …" doesn't match musical preference.
- **Grid shape:** still 8 columns × up to 8 rows, flat sequence. Empty display-cells render blank space.
- **Cell controls match Phrase view:** boolean toggle, scalar drag-to-fill, indexed picker (tap-cycle / long-press-popover). Chord is read-only. Per-cell subdivision modals (`.bars` / `.steps` / `.curve`) deferred — Live view is for live manipulation of `.single` values.
- **Save as Phrase:** snapshot of the currently-active phrase's cells into a new fully-independent phrase. No linking; edits to either don't affect the other.
- **Transport mode default:** `.free`. A new project has no song yet.
- **Free mode with no phrases:** document always has at least one phrase; engine plays it indefinitely.
- **Live edits in song mode:** persistent — they stick to the phrase they were edited in. Temporary overlay state deferred (a separate live-overlay data-layer would let edits auto-revert at phrase boundary).
- **Drum per-tag Pattern layer:** deferred. Pattern layer on a drum track picks ONE pattern index for the whole track; per-tag sub-patterns are a bigger data-model change.
- **Mixer-bus mute:** remains Mixer's concern. Per-track and per-tag mute belong with the phrase / voicing concept, which is Live view's domain.
- **Muted-tag engine enforcement:** Task 6 wires the router/controller to drop events whose `voiceTag` is in the phrase's muted-tag set. No audio-side work needed; the filter is on the note-stream, pre-destination.
