# Live Performance View + Transport Modes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the app a **Live view** — an 8×8 matrix of tracks (same slot layout as the Tracks matrix) where each cell shows the current value of one layer (Pattern / Mute / Volume / Intensity / …) for that track, live-editable while the transport is running. A layer selector at the top switches which axis is visible. Cell controls match the Phrase view's (toggle for booleans, drag-to-fill for scalars, step-value-picker for indexed layers). Drum tracks can be expanded inline to mute or re-pattern individual voice tags. A "Save as Phrase" action captures the current grid state as a new phrase in the project. Paired with this, transport gains two modes: **song** (follows `document.phrases` list) and **free** (stays on the current phrase, letting Live-view edits drive the sound directly).

**Architecture:** Reuses the existing phrase/layer/cell data model entirely — no new data types. The view operates on `document.phrases[currentPhraseIndex].cells[(trackID, layerID)]`, which is the same path the Phrase view writes to. Layout is `LazyVGrid` with 8 columns × 8 rows = 64 cells, matching `TracksLayout.slotCount`. Cell contents come from `TrackCellFor(track:layer:phrase:)` — a polymorphic view that renders the right control for the layer's value type. Drum expansion: when the active layer is `Mute` (or `Pattern`) and the cell's track is a drum, the cell grows vertically to show N sub-cells, one per `track.voicing.destinations` key. Transport mode is `EngineController.transportMode: TransportMode = .song | .free`; in `.free` the engine pins `currentPhraseIndex` and never advances; in `.song` it advances at phrase-boundary per the existing song-driver logic (stubbed to always return index 0 until the song plan lands). "Save as Phrase" snapshots the active phrase's cells into a brand-new `Phrase` appended to `document.phrases`, prompts for a name, and highlights the new phrase in Song view.

**Tech Stack:** Swift 5.9+, SwiftUI (LazyVGrid, DragGesture, ContextMenu), Foundation, XCTest.

**Parent spec:** `docs/specs/2026-04-18-north-star-design.md` — §"UX surfaces" (Perform view's neighbour), §"Track types, patterns, and phrases" (phrase cell model reused), §"Pipeline layer" (mute is a concrete macro row). Live view isn't explicitly listed in the spec yet — this plan's §"Spec delta" at the bottom appends a short paragraph.

**Environment note:** Xcode 16. All `xcodebuild` invocations prefix `DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer`.

**Status:** <STATUS_PREFIX> <COMPLETED_MARKER> TBD. Tag `v0.0.8-live-view` at TBD.

## Task 4: `LiveCell` polymorphic control

**Scope:** The per-cell view. Takes a `(track, layer, cellValue)` trio and returns the right control for the layer's value type. Cell edits write back to `document.phrases[currentIndex].cells[(track.id, layer.id)]` via a closure.

**Files:**
- Create: `Sources/UI/LiveCell.swift`
- Create: `Sources/UI/LiveCellControls/BooleanToggleCell.swift`
- Create: `Sources/UI/LiveCellControls/ScalarDragCell.swift`
- Create: `Sources/UI/LiveCellControls/IndexedPickerCell.swift`
- Create: `Sources/UI/LiveCellControls/ChordCell.swift` (read-only)
- Create: `Tests/SequencerAITests/UI/LiveCellTests.swift`

**Type:**

```swift
struct LiveCell: View {
    let track: StepSequenceTrack
    let layer: Layer
    let cellValue: CellValue
    let onEdit: (CellValue) -> Void

    var body: some View {
        switch layer.valueType {
        case .boolean:
            BooleanToggleCell(value: cellValue.asBool, onToggle: { onEdit(.single(.bool($0))) })
        case .scalar(let range):
            ScalarDragCell(value: cellValue.asDouble, range: range, onChange: { onEdit(.single(.double($0))) })
        case .patternIndex:
            IndexedPickerCell(value: cellValue.asInt, maxIndex: 15, onPick: { onEdit(.single(.int($0))) })
        case .chord:
            ChordCell(chord: cellValue.asChord)
        case .enumTag(let tags):
            IndexedPickerCell(value: ..., labels: tags, onPick: ...)
        }
    }
}
```

**BooleanToggleCell:** tap toggles. Renders a filled square when true; outlined square when false. `.accessibilityIdentifier("live-cell-bool-\(trackID)-\(layerID)")`.

**ScalarDragCell:** vertical drag gesture. Drag up = increase, drag down = decrease, clamped to range. Renders a column with a fill height proportional to the current value. Tooltip shows numeric. `.accessibilityIdentifier("live-cell-scalar-\(trackID)-\(layerID)")`.

**IndexedPickerCell:** tap cycles through values; long-press opens a mini-popover with all slot indices. Shows the current index. `.accessibilityIdentifier("live-cell-index-\(trackID)-\(layerID)")`.

**ChordCell:** read-only. Shows "Cmaj7" or "Gmin" text derived from the current chord-context. `.accessibilityIdentifier("live-cell-chord-\(trackID)-\(layerID)")`.

**Tests (per cell variant):**

1. BooleanToggleCell: tap → `onToggle(true)`; tap again → `onToggle(false)`.
2. ScalarDragCell: drag +10pt (vertically) with range 0–1 and starting value 0.5 → `onChange` called with ~0.6 (exact mapping depends on drag-sensitivity constant; assert within tolerance).
3. IndexedPickerCell: tap → `onPick(current + 1 mod maxIndex)`; long-press opens popover (state-inspection test).
4. ChordCell: shows the chord's name; read-only.
5. `LiveCell` delegates to the correct sub-view per layer value type.

- [ ] Tests for each sub-cell
- [ ] Implement sub-cells + LiveCell dispatch
- [ ] Green
- [ ] Commit: `feat(ui): LiveCell + per-value-type controls (toggle, drag, picker, chord)`

## Task 5: `LivePerformanceView` — 8×8 grid + layer selector + active-phrase binding

**Scope:** The main view. 8×8 grid of cells, one per populated track slot (empty slots stay empty). Layer selector at the top. A "Save as Phrase" button bottom-right. Reads/writes to `document.phrases[engineController.currentPhraseIndex].cells`.

**Files:**
- Create: `Sources/UI/LivePerformanceView.swift`
- Modify: `Sources/UI/ContentView.swift` — wire `.live` to this view
- Create: `Tests/SequencerAITests/UI/LivePerformanceViewTests.swift`

**View shape:**

```swift
struct LivePerformanceView: View {
    @Binding var document: SeqAIDocument
    @ObservedObject var engineController: EngineController
    @State private var activeLayerID: LayerID = .pattern
    @State private var isSaveAsPhrasePresented = false

    private let columns: [GridItem] = Array(
        repeating: GridItem(.flexible(), spacing: 8),
        count: TracksLayout.columnCount
    )

    private var activePhrase: Phrase? {
        document.model.phrases[safe: engineController.currentPhraseIndex]
    }

    var body: some View {
        VStack(spacing: 12) {
            LayerSelectorBar(
                layers: document.model.layers,
                activeLayerID: $activeLayerID
            )

            LazyVGrid(columns: columns, spacing: 8) {
                ForEach(0..<TracksLayout.slotCount, id: \.self) { slotIndex in
                    if let track = document.model.track(atSlot: slotIndex),
                       let phrase = activePhrase,
                       let layer = document.model.layer(id: activeLayerID)
                    {
                        LiveCell(
                            track: track,
                            layer: layer,
                            cellValue: phrase.cellValue(for: track.id, layer: layer.id),
                            onEdit: { newValue in
                                document.model.setCell(
                                    phraseID: phrase.id,
                                    trackID: track.id,
                                    layerID: layer.id,
                                    value: newValue
                                )
                            }
                        )
                        .aspectRatio(1.0, contentMode: .fit)
                    } else {
                        Color.clear
                            .frame(maxWidth: .infinity, maxHeight: .infinity)
                            .aspectRatio(1.0, contentMode: .fit)
                    }
                }
            }
            .padding()

            HStack {
                Spacer()
                Button("Save as Phrase") {
                    isSaveAsPhrasePresented = true
                }
                .accessibilityIdentifier("save-as-phrase")
            }
            .padding(.horizontal)
        }
        .sheet(isPresented: $isSaveAsPhrasePresented) {
            SaveAsPhraseSheet(
                isPresented: $isSaveAsPhrasePresented,
                suggestedName: "Phrase \(document.model.phrases.count + 1)"
            ) { name in
                document.model.appendPhrase(copyingCellsFrom: engineController.currentPhraseIndex, name: name)
            }
        }
        .accessibilityIdentifier("live-performance-view")
    }
}
```

**Tests:**

1. Renders 64 cells; populated slots show `LiveCell`, empty slots stay blank.
2. Layer selector switches — cells re-render against the new layer.
3. Editing a cell calls `document.model.setCell(...)`.
4. "Save as Phrase" button triggers the sheet.
5. Save flow: appending a phrase adds a new entry to `document.model.phrases` with cells copied from the current active phrase.
6. Accessibility identifiers present: `"live-performance-view"`, `"save-as-phrase"`.

## Open questions resolved for this plan

- **Grid shape:** exactly the 8×8 slot layout of Tracks matrix. One square cell per slot. Empty slots render blank (no placeholder — different from the Tracks matrix's "+" empty cells; Live view is about tracks-that-exist).
- **Cell controls match Phrase view:** boolean toggle, scalar drag-to-fill, indexed picker (tap to cycle; long-press for list). Chord is read-only (it displays chord-context broadcast). Deferred: per-cell subdivision modals (`.bars` / `.steps` / `.curve`) — Live view is for live manipulation, so `.single` suffices. If the user wants subdivision they use the Phrase view.
- **Drum expansion:** Mute layer only for MVP. Pattern-layer per-tag expansion (different patterns per tag on a drum track) requires a per-tag pattern-index model that doesn't exist yet — a bigger piece of work.
- **Save as Phrase:** snapshot of the currently-active phrase's cells into a new phrase. No diff, no linking back — the new phrase is fully independent after creation. The user can edit either phrase without affecting the other.
- **Transport mode default:** `.free`. A new project has no song yet, so `.song` would do nothing useful; `.free` lets the engine play a single phrase directly, matching Elektron / Octatrack's default behaviour when no pattern chain is set.
- **Free mode with no phrases:** the document always has at least one phrase (the default phrase created on document init). Engine plays it indefinitely.
- **Live edits in song mode:** persistent. Editing cell values while song mode is playing writes into whichever phrase is currently active; when the song moves to the next phrase, the edit remains attached to the phrase-it-was-edited-in. To get "temporary overlay" behaviour, we'd need a separate live-overlay-state that wraps the underlying phrase — deferred.
- **Drum per-tag Pattern layer:** deferred (noted above). MVP = Pattern layer on a drum track picks ONE pattern index for the whole track; all tagged voices in that pattern fire according to the pattern's contents.
- **Mixer-bus mute:** remains Mixer's concern. Live view's per-track and per-tag mute is about performance; Mixer's per-bus mute is about routing. They don't overlap.

And to §"Vocabulary":

> - **Transport mode** — a runtime setting on the engine: `.song` (playback follows `project.phrases` order) or `.free` (playback stays on one phrase, Live view drives). Toggled from the transport bar. Default `.free` until a song is authored.
